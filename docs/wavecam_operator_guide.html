01 Start

The clean start path

Start from known state. The phone can show video while safety is killed; tracking motion only resumes after the latch is cleared and auto PTZ is started.

Connect the rig.Camera RJ45 to Orin camera LAN. Phone can be USB-C tethered to the Orin or on the same Wi-Fi/hotspot path. Open WaveCam on the phone.
Confirm API route.Connect tab now has two fields: USB tether http://172.20.10.8:8088/api/v1 and Wi-Fi/hotspot http://192.168.1.155:8088/api/v1. The app uses the first route that answers and shows USB, WIFI, CUSTOM, or OFFLINE.
Check Live.Live should show video. Top status should say USB or WIFI when the app can reach the backend.
If STOP LATCHED appears, hold resume.This clears KILL only. It does not automatically restart tracking.
Open PTZ and tap Start Auto.That requests autonomous tracking ownership. The owner should move to testbed and state should become SEARCHING or TRACKING.
Do first
Use Emergency Stop if the camera is moving the wrong direction or locking the wrong object.
Then
Use Stop PTZ to hold the camera still without latching global safety.
Then
Re-aim with joystick if needed. Joystick release is temporary and hands control back unless Stop PTZ is held.
Finally
Use Start Auto to let the tracker reacquire the target.

03 Controls

What each PTZ control means

The controls are deliberately separated by authority. Safety latch, manual hold, manual nudge, and autonomous tracking are different states.

■
Emergency Stop

Global safety latch. Sends pan/tilt stop and zoom stop, drops ownership, and blocks future motion until Resume.

■
Stop PTZ

Manual hold. Camera stays stopped and the Stop PTZ button remains highlighted until Start Auto is pressed.

▶
Start Auto

Requests autonomous tracking ownership. It will refuse while KILL is latched; resume first.

◇
Joystick and zoom

Temporary manual takeover. Release stops movement and restores autonomous owner when there is one.

04 Readouts

Do not infer. Read the state.

If the camera appears wrong, use the state labels before changing settings. They tell you whether video, safety, PTZ authority, and tracking agree.

Connection

USB, WIFI, or CUSTOM means the app can reach the backend on that route. OFFLINE means neither configured route is answering.

Safety

KILLED means motion is blocked. The live feed may continue because video preview is not the same as motion authority.

Owner

idle no PTZ writer. manual operator hold/nudge. testbed autonomous tracker owns PTZ.

Command

stop means no pan/tilt command. p#/t# means pan and tilt velocities are being commanded.

Tracking

SEARCHING no lock. TRACKING lock active. Confidence is the fusion score, not proof the right object is selected.

Live feed

Video comes from /api/v1/preview.mjpeg. A blank feed with ORIN connected is a preview/rendering problem, not necessarily a camera-control problem.

05 Tuning

Tracking trigger and weights

The Orin web UI at http://192.168.1.155:8088/ exposes the full live tuning surface. The native Tune tab exposes the common hot controls, restart-only key list, and confirmed Restart WaveCam action. Persistent defaults live in /data/projects/gimbal/wavecam/config.orin.servo.yaml.

Key Current default Effect When to change
color.preset orange_red HSV preset used by the color detector. Switch first when the field marker changes color. Supported presets: orange_red, orange, blue, green, yellow, pink.
fusion.require_person false Requires person detection before color can lock. Turn on when orange objects like gear, skis, boards, or signs steal the lock.
fusion.match_dist 120 Maximum distance between color blob and person box for a validated match. Lower it if nearby objects merge into the target; raise only if person/color alignment is too strict.
fusion.person_aim_y 0.50 Vertical aim point inside the YOLO person box. Lower toward 0.35 for upper-body framing. Lower further only after checking tilt response.
fusion.lock_threshold 0.60 Score required to enter TRACKING. Raise it when the system locks too easily. Lower it only if it cannot acquire the correct shirt.
fusion.unlock_threshold 0.35 Score below which lock drops. Raise it to drop bad locks faster. Keep it below lock_threshold.
color.min_area 60 Smallest color blob accepted. Raise it when small orange noise causes false detections.
color.max_area 200000 Largest color blob accepted. Lower it if glare, walls, or a large warm surface floods the mask.
color.morph_kernel 5 Mask cleanup size. Raise slightly for noisy speckles; lower if the real marker gets eroded.
detector.conf 0.35 YOLO person confidence threshold. Raise for fewer person false positives; lower for harder distance/acquisition conditions.
detector.person_class 0 YOLO class used as the validator trigger. Keep 0 for person tracking. Other classes are exposed for testing, not proven field defaults.
detector.every_n 3 Runs YOLO every N frames and reuses boxes between runs. Lower for faster validation at higher compute cost; raise if CPU/GPU load becomes a problem.
ptz.deadzone 0.05 Center hold region where pan/tilt should stop. Raise when the camera jitters near center. Lower only when it feels too lazy around center.
ptz.ff_gain 0.00 Optional feed-forward lead on moving targets. Leave off until normal tracking is stable. Increase carefully if the camera lags a fast target.
ptz.ff_deadzone_mult 1.5 Suppresses feed-forward inside/near the deadzone. Raise if small detection noise still causes left/right/up/down twitching.
ptz.max_pan_speed / ptz.max_tilt_speed 10 / 12 Upper pan/tilt velocity limits for the servo. Lower when tracking overshoots. Raise only after deadzone and target selection are correct.
ptz.cinematic_zoom_enabled / ptz.zoom_target_frac false / 0.50 Person-box auto-zoom that keeps the subject near a chosen frame size. Default off. Enable only after pan/tilt tracking is stable and the lock is on a YOLO person box. Manual zoom suppresses cinematic zoom briefly without stopping pan/tilt tracking.

Cinematic Zoom enable/test flow

  1. Confirm the app is connected on USB or WIFI and Live shows MJPEG video.
  2. Open PTZ, tap Start Auto, and verify backend state first: owner should resolve to autonomous tracking and CMD should reflect backend pan/tilt state.
  3. Open Tools > Tune. If the Cinematic Zoom card is not visible, the backend did not expose the zoom config keys; stop and verify /api/v1/config.
  4. Enable Cinematic zoom and leave Subject size near 0.50 for the first pass.
  5. Watch Live. Zoom should move only when tracking is locked on a YOLO person box. It should hold on color-only frames, no-lock frames, and inside the zoom deadband.
  6. Nudge manual zoom once. Expected result: auto-zoom suppresses briefly, but pan/tilt tracking continues.
  7. If framing twitches, locks the wrong subject, or zoom fights the operator, disable Cinematic Zoom and retest pan/tilt before changing zoom thresholds.

Hot tuning call

$ curl -X POST http://127.0.0.1:8088/api/v1/config/hot \
-H "Content-Type: application/json" \
-d '{"patch":{"color.preset":"blue","fusion.require_person":true,"ptz.ff_deadzone_mult":1.8}}'

Persistent config

Use the YAML path for defaults that should survive service restart. Hot config is for live testing without restart; it is not persisted automatically yet.

Scheduled restart

$ curl -X POST http://127.0.0.1:8088/api/v1/system/restart \
-H "Content-Type: application/json" \
-d '{"reason":"applied structural config","confirm_moving":true}'

This returns 202, stops PTZ, marks the session RESTARTING, and then asks systemd to restart wavecam.service. If PTZ is active and confirm_moving is not true, the API refuses the request.

Current gaps

  1. Hot tuning does not write back to YAML automatically.
  2. Restart-only keys still require editing persistent config or the Orin web UI before using the confirmed restart path.
  3. Cinematic Zoom is backend-wired and default off. Use the enable/test flow above before making it a persistent default.
  4. The service restart timeout was fixed with a VISCA brake plus fast systemd stop; this is validated, but it is not a graceful TensorRT thread teardown.
06 Recovery

Common failures and the fastest safe response

Use Stop PTZ when you want the camera held still but the system still reachable. Use Emergency Stop when physical motion itself is wrong or unsafe.

Camera tracks the wrong object.Hit Stop PTZ or Emergency Stop. If orange gear is stealing lock, enable fusion.require_person and consider raising fusion.lock_threshold.
No Live video.Check Local Network permission, active route (USB/WIFI), both Connect tab URLs, and whether /api/v1/preview.mjpeg returns bytes.
PTZ buttons do nothing.If safety is KILLED, resume first. If owner is not manual/testbed, read the owner field before repeating commands.
Tilt direction is wrong.Use ptz.invert_tilt in the Orin web tuning UI, hot config, or YAML. Current Orin servo config default is false.
Camera twitches near center.Raise ptz.deadzone or ptz.ff_deadzone_mult. Leave ptz.ff_gain at 0.00 unless you are deliberately testing lead compensation.

Wrong target reset

  1. Tap Emergency Stop if the camera is moving unexpectedly.
  2. Hold Resume only when you are ready for commands again.
  3. Use PTZ joystick to put the correct target near frame center.
  4. Tap Stop PTZ if you want to hold that framing.
  5. Apply stricter tuning if false locks repeat.
  6. Tap Start Auto and verify owner, state, confidence, and the live frame.
