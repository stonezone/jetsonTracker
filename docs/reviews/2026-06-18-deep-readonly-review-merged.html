B. Open findings

Critical

CriticalCK·B2Estimator covariance update drops all off-diagonal P terms (non-PSD)estimator.py:414-432✓ VERIFIED
Why
Update must be P ← P − (K⊗h)P; element [i][j] = Σ_k K[i]h[k]P[k][j]. Line 432 does P[i][j] −= K[i]*h[j]*P[j][j] — only the k=j diagonal term, discarding all cross terms. Asymmetric; can lose PSD under repeated updates. Kimi adds: the GPS branch (:301-305) uses the full/correct update — only the vision scalar path is broken.
Impact
Shadow-only today (never commands). Blocker before closed-loop/COMMAND mode.
Fix
Full product then symmetrize, or Joseph form (I−KH)P(I−KH)ᵀ+KRKᵀ; add a PSD-under-large-innovation unit test.

Why
Preflight raises FileNotFoundError, but Pipeline.__init__ catches broad Exception, prints [pipeline] YOLO disabled, sets detector=None. API stays up; vision dead; no /events, no detector_loaded flag, no .pt fallback.
Fix
Record a detector_failed event; surface detector_loaded:false on /status+/health; optionally fall back to bundled .pt.

Why
Every API transition (and the arbiter thread) does release(old) then request(new) as two unlocked ptz_owner calls. Between them the vision loop can observe IDLE and steal the owner; operator's manual claim fails owner_busy. 4d52a07 fixed only the separate zoom-drops-STOP bug.

Single shared transition lock; atomic PtzOwner.transition(from,to); guard the pipeline's release→request pair.
