MediumCovariance update uses diagonal only — P drifts non-symmetric / non-PSD
orin/wavecam/wavecam/estimator.py:429-432
whatThe update does P[i][j] -= K[i] * h[j] * P[j][j] (diagonal for every column). The Joseph form requires K[i] * (h·P[:,j]) — the full column. Off-diagonal covariances are updated with wrong values; P is not guaranteed symmetric/PSD.
whyOver many ticks bearing_std_deg (reads P[0][0],P[1][1]) can drift and gain K can go erratic. Shadow-only today; corrupts the fusion if flipped live.
fix
# P -= K * (h @ P) — full row, not just the diagonal
hP = [sum(h[k] * self._P[k][j] for k in range(4)) for j in range(4)]
for i in range(4):
    for j in range(4):
        self._P[i][j] -= K[i] * hP[j]

def calibrate_pan_two_point(self, enc1, bearing1, enc2, bearing2):
    dbear = normalize_180(bearing2 - bearing1)
    if abs(dbear) < 1e-6:
        raise ValueError("pan calibration points have ~equal bearing")
    scale = (enc2 - enc1) / dbear
    if scale <= 0:
        raise ValueError(f"pan scale non-positive ({scale:.3f} enc/deg) — re-capture panning in the +bearing direction.")
    self.pan_enc_per_deg = scale
    self.pan_anchor_enc = enc1
    self.pan_anchor_bearing = bearing1
