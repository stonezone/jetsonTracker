Field Test Checklist

Current · goal: validate control + vision + safety + observability, then prove GPS tracking at range.
Safety first: the KILL / Emergency-Stop button in the app must stay reachable the whole session. The estimator never moves the camera — if the camera does anything unexpected, hit STOP.

B — Driveway shakedown (close range)
Why driveway first: prove control + vision + safety + observability before the water test. GPS bearing accuracy is not meaningful here (≈27° error at 10 m; heading needs a stationary target ≥50 m). Keep gps.drive_zoom and fusion.gps_bearing_cue_enabled OFF.
Base Wio has a battery installed — let it acquire on battery outside, then plug into Orin USB. Power on the tracker Wio (worn outside, clear sky).
reader_alive true; gps.source = direct_lora; base fix arrives within ~2 min; tracker sats > 0 and target_age_sec stays low.
target_age_sec climbing forever = no LoRa packets reaching the base. Check tracker battery/antenna, base LED, and that both Wios run matching direct-LoRa firmware.

authority.base_locked: true; authority.base_drift_state: locked; after validate → confirm, authority.calibration_valid: true.
Calibration valid stays false → check base fix, level step, and that heading capture completed.
Set tracking.mode: vision_only. Start AUTO. Stand ~10–30 m from the camera wearing the orange rashguard.
tracking.locked: true, tracking.state: TRACKING, authority.owner: vision_follow; camera follows as you walk laterally.
Locks wrong object → turn on Show mask, tune color preset, or raise fusion.require_person.
Emergency Stop.
Camera stops immediately; safety.killed: true; authority.owner: idle; manual + auto both blocked. Resume → it re-searches.
Camera keeps moving → use physical power. Do not proceed until KILL is verified.
Resume, start AUTO, then joystick nudge.
authority.owner: manual while nudging; release → auto-releases after the deadman (~0.8 s) back to search/auto.
Owner sticks at manual → check deadman timeout or ptz.command_min_interval.
With base locked, physically nudge/drag the tripod a few metres.
authority.base_drift_state goes locked → suspect → unlocked and GPS authority is withheld. Re-CALIBRATE to clear.
No state change → confirm gps.base_drift_enabled: true; if it trips on jitter, raise gps.base_drift_threshold_m (restart to change thresholds).
C — Water test (50–300 m)
Before entering the water, place the tracker at a stationary target ≥50 m away. Aim the camera at it. Calibrate tab → capture heading, validate, confirm.
authority.calibration_valid: true; GPS chip shows live distance + bearing; authority.base_drift_state: locked.
Capture blocked = no live GPS bearing. Check tracker fix and base stability.
Set tracking.mode: gps_only. Walk or paddle 50–300 m out with the tracker Wio.
Camera slews to GPS bearing and follows coarsely; authority.owner: gps_tracker.
No slew → verify calibrated + base_locked + GPS fresh. Points wrong → recalibrate heading at ≥50 m.
Set tracking.mode: auto. Move through close, mid, and far ranges; let vision and GPS trade ownership.
Smooth handoff: vision owns close range, GPS owns far range, camera holds (not coasts) during dropouts.
Hunting or freezes → note range/behavior, hit STOP, adjust lock/unlock thresholds or GPS boost in Tune.
One at a time, after each previous step is solid: detector.tracker (restart-required; e.g. bytetrack.yaml), fusion.gps_bearing_cue_enabled, fusion.gps_roi_enabled, gps.drive_zoom.
Each feature improves lock continuity or zoom framing without destabilizing pan/tilt.
Disable the last feature and retest pan/tilt stability before changing its knobs.
D — Wrap up
Hit STOP / KILL. Pull recordings, logs, and any watch session JSONL you recorded.
Clips on the Orin Media tab; watch sessions transferred to the paired iPhone.
If the service is wedged, Tools → Restart WaveCam.
Intentionally OFF by default for the first field run: detector.tracker (None), fusion.gps_bearing_cue_enabled (false), fusion.gps_roi_enabled (false), gps.drive_zoom (false). Base drift is ON (gps.base_drift_enabled: true) but fail-safe — only confirmed drift withholds GPS.

Troubleshooting

App can't reach the Orin
Toggle tether → hotspot. If both fail and you have cell signal, check Connect tab routes or restart the Orin Wi-Fi.
GPS stale mid-session
Tracker lost fix or base lost packets. Check tracker LED (fast flash = no fix). If the base serial reader died, restart wavecam.service.
Service wedged / app shows stale everything
Tools → Restart WaveCam. The watchdog also auto-recovers a wedged service within 2 min.
Camera won't move at all
Check E-stop state in the app first — a latched stop looks identical to a fault.
Heading capture keeps failing
Needs a live tracker fix, base lock, and a stationary target ≥50 m away. Re-do CALIBRATE after any tripod move.
Base drift trips too easily / not at all
gps.base_drift_enabled is hot. Threshold/window knobs are restart-only; default threshold_m=4.0, min_trend_m=2.0.
